# GitHub Desktop 常用操作速查表 📋

## 一、核心操作流程

### 1. 克隆仓库到本地
- **操作**：`File` → `Clone repository...`
- **用途**：将 GitHub 网页端的仓库复制到本地电脑，实现双向同步。
- **注意**：选择空文件夹作为本地路径，避免文件冲突。

### 2. 查看本地文件
- **操作**：点击 `Show in Explorer` 或使用快捷键 `Ctrl + Shift + F`
- **用途**：打开本地仓库文件夹，直接编辑或添加文件（如 Excel 模板、文档）。

### 3. 提交本地修改
- **操作**：在左侧 `Changes` 面板填写提交信息（Summary）→ 点击 `Commit to main`
- **用途**：将本地修改保存到 Git 版本历史。

### 4. 推送到远程仓库
- **操作**：点击右上角 `Push origin`
- **用途**：将本地提交的修改同步到 GitHub 网页端，完成更新。

### 5. 拉取远程更新
- **操作**：点击 `Fetch origin` → 点击 `Pull origin`
- **用途**：将 GitHub 网页端的最新修改同步到本地，保持版本一致。

---

## 二、常见场景与解决方案

| 场景 | 操作步骤 |
| :--- | :--- |
| 本地修改后同步到网页 | 1. 编辑本地文件 → 2. 填写提交信息 → 3. `Commit to main` → 4. `Push origin` |
| 网页修改后同步到本地 | 1. 点击 `Fetch origin` → 2. 点击 `Pull origin` |
| 克隆时提示文件夹非空 | 1. 选择空文件夹或新建文件夹 → 2. 重新点击 `Clone` |
| 提交后发现错误 | 1. 在 `History` 面板找到错误提交 → 2. 右键选择 `Revert this commit` |

---

## 三、界面关键区域说明

- **左上角仓库选择器**：切换当前管理的仓库。
- **左侧 Changes 面板**：显示本地未提交的修改，填写提交信息。
- **顶部 Branch 菜单**：创建、切换或合并分支（进阶操作）。
- **右上角 Fetch/Push 按钮**：同步本地与远程仓库的核心入口。

## 四、快捷键速记

| 功能 | 快捷键 |
| :--- | :--- |
| 打开本地仓库文件夹 | `Ctrl + Shift + F` |
| 在浏览器中打开仓库 | `Ctrl + Shift + G` |
| 提交到主分支 | `Ctrl + Enter` |
| 推送修改 | `Ctrl + P` |

---

## 五、版本同步原则

1.  **先拉取，再推送**：在本地修改前，先 `Pull origin` 确保本地是最新版，避免冲突。
2.  **提交信息清晰**：每次提交都写清楚修改内容，方便日后回溯。
3.  **及时同步**：修改完成后尽快 `Push origin`，确保网页版和本地版一致。
