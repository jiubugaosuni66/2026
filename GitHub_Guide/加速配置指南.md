GitHub 访问加速与稳定配置指南
文档说明
本文档用于解决 GitHub 在国内访问慢、加载失败、无法提交代码等问题。核心方法是通过修改系统 hosts 文件，将 GitHub 的域名解析到最快的 IP 地址。
适用系统
Windows 10 / Windows 11
操作步骤
第一步：以管理员身份打开记事本
使用快捷键 Win + S 召唤系统搜索框。
输入 记事本。
右键点击 搜索结果中的 “记事本”，选择 以管理员身份运行。
若弹出用户账户控制提示，点击 是。
第二步：打开 hosts 文件
在记事本中，点击顶部菜单栏 文件 (F) -> 打开 (O)。
在文件路径栏输入：C:\Windows\System32\drivers\etc，并回车。
在 “打开” 窗口的右下角，将 文件类型 从 “文本文档 (.txt)” 改为 **“所有文件 (.*)”**。
选中列表中的 hosts 文件，点击 打开。
第三步：配置加速 IP
将光标移动到 hosts 文件的最末尾，换行。
复制下方 【推荐 IP 配置】 中的内容，粘贴到文件末尾。
点击记事本菜单栏 文件 (F) -> 保存 (S)，然后关闭记事本。
第四步：刷新 DNS 缓存（关键）
使用快捷键 Win + R，输入 cmd，按回车打开命令提示符。
在黑色窗口中输入以下命令，按回车执行：
bash
运行
ipconfig /flushdns
看到提示 “成功刷新 DNS 解析缓存” 即表示操作完成。
第五步：验证效果
关闭所有浏览器窗口，重新打开访问 https://github.com，页面加载速度应明显提升。

常见问题解决
保存时提示 “拒绝访问”：
原因：没有以管理员身份运行记事本。
解决：关闭记事本，重新按照第一步操作。
修改后依然无效：
尝试更换一组备用 IP。
清除浏览器缓存（快捷键 Ctrl + Shift + Delete）。
重启电脑。
如何恢复原状：
打开 hosts 文件，删除我们添加的几行代码，保存并刷新 DNS 即可。
